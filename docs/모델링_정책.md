# 모델링 정책

## Primary model

현재 primary model은 다음 방향입니다.

```text
voltage waveform + frequency -> measured magnetic field waveform
```

기존의 `전압 -> 전류 -> 자기장` 중심 설명은 main modeling direction이 아닙니다.

## Physical Target

Physical Target은 사용자가 원하는 실제 자기장 파형입니다.

현재 정책:

- target field shape: rounded triangle
- target field pp: 100pp fixed
- Bz 또는 현재 앱의 main field axis를 우선 사용

## Support Reference

Support Reference는 실험 데이터에서 가져온 비교/보간 기준입니다. Support Reference는 Physical Target이 아니며, UI에서 target처럼 보여서는 안 됩니다.

## Predicted Output

Predicted Output은 현재 command를 넣었을 때 예상되는 field output입니다. Physical Target과 Support Reference와 구분해서 봐야 합니다.

## support/input waveform family

support/input waveform family는 실험 support를 고르는 선택입니다. 이 선택은 physical target shape를 바꾸지 않습니다.

예:

- support/input waveform family: sine
- physical target: rounded triangle / 100pp fixed

## excluded from main shape selection

다음 값은 main shape selection 근거가 아닙니다.

- current
- gain
- hardware
- LCR

이 값들은 debug/reference 또는 장비 제한 참고용입니다.

## End markers

finite-cycle 결과에서는 세 marker를 분리해서 봅니다.

- target_end: 사용자가 원하는 physical target 종료 시점
- command_end: target을 만들기 위해 command를 언제까지 넣는지
- predicted_settle_end: field가 실제로 언제 정착하는지

세 marker를 하나로 섞으면 안 됩니다.

## 1.75 cycle 정책

1.75 cycle은 exact finite-cycle support data가 있을 때 사용합니다. 0.75 cycle과 동일시하지 않습니다.

## retune policy

retune acceptance threshold와 핵심 retune 정책값은 casual하게 바꾸지 않습니다. 데이터 검수 단계에서는 모델링/solver를 우회하지 말고 Raw Waveforms에서 문제를 확인한 뒤 재측정 또는 quarantine 여부를 결정합니다.
