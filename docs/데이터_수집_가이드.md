# 데이터 수집 가이드

## 파일명 규칙

continuous 데이터:

```text
continuous_{waveform}_{freq}Hz.csv
```

예:

```text
continuous_sine_1Hz.csv
continuous_triangle_0.25Hz.csv
```

finite-cycle 데이터:

```text
finite_{waveform}_{freq}Hz_{cycle}cycle.csv
```

예:

```text
finite_sine_1Hz_1.25cycle.csv
finite_triangle_2Hz_1.75cycle.csv
```

## 고정 실험 조건

- DAQ output: `±5V`
- DAQ PP: `10V`
- DCAMP Gain: `100%`

새 데이터 파일명에는 current/App이 없을 수 있습니다. 앱은 current/App이 없어도 Raw Waveforms label을 만들 수 있어야 합니다.

## 권장 수집 항목

- waveform family: sine 또는 triangle
- freq_hz
- cycle_count: finite-cycle일 때 필수
- duration
- source file
- raw/corrected 여부
- internal id
- sampling rate 또는 dt

## baseline / tail

finite-cycle 데이터는 시작 전 pre-baseline과 종료 후 post-tail을 충분히 포함해야 합니다.

- pre-baseline: 구동 전 0 상태 확인용
- active 구간: 실제 command가 들어가는 구간
- post-tail: command 종료 후 자기장이 가라앉는지 확인하는 구간

post-tail이 너무 짧으면 predicted_settle_end와 tail residual 검수가 어렵습니다.

## sampling rate

sampling rate는 파형의 상승/하강, spike, truncation을 볼 수 있을 만큼 충분히 높아야 합니다. 파일마다 sampling rate가 다르면 Raw Waveforms에서 duration, row count, sampling rate를 먼저 확인하십시오.

## sign convention note

코일 방향, 센서 축, DAQ polarity가 바뀌면 같은 physical target이라도 부호가 달라질 수 있습니다. 재측정 시 operator note에 polarity, coil 방향, sensor 축 기준을 남기는 것을 권장합니다.
